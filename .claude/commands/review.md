# Review Code

Review the current changes before committing.

Tasks:

1. Run:

```bash
git status
git diff
```

2. Review for:
   - Bugs
   - Type safety issues
   - Edge cases
   - Unused code
   - Dead code
   - Unused imports
   - Performance issues
   - Accessibility issues
   - Security concerns

3. Verify project rules are followed.

4. Suggest improvements ranked by:
   - Critical
   - Recommended
   - Optional

5. Do not modify files unless explicitly asked.

6. Provide a concise review summary.

7. Suggest a conventional commit message if changes are ready.
