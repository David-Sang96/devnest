# Cleanup Code

Improve code quality without changing behavior.

Tasks:

1. Scan project files.

2. Remove:
   - Unused imports
   - Unused variables
   - Dead code
   - Duplicate code
   - Obsolete comments

3. Simplify:
   - Complex conditionals
   - Repeated logic
   - Over-engineered patterns

4. Respect project rules:
   - Prefer simple solutions
   - No premature abstractions
   - No unnecessary optimization

5. After cleanup:
   - Summarize changes
   - Show affected files
   - Verify behavior remains unchanged

Rules:

- Do not add features.
- Do not change functionality.
- Do not introduce abstractions without justification.
